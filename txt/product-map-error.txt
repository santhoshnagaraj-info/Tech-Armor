I found the issue. It's in your **service** and **hook**.

### Your API response

Your console showed:

```json
{
  "success": true,
  "message": "Success",
  "data": [
    { ... },
    { ... }
  ]
}
```

But your hook expects `getProducts()` to return only:

```ts
[
  { ... },
  { ... }
]
```

Instead, your service currently returns the **entire response object**.

### Current code

```ts
// product.service.ts
export const getProducts = async () => {
  const response = await api.get("/products");

  return response.data;   // ❌ returns { success, message, data }
};
```

Then your hook does

```ts
getProducts().then(setProducts);
```

which becomes

```ts
setProducts({
  success: true,
  message: "Success",
  data: [...]
});
```

Now `products` is **not an array**, so

```ts
products.map(...)
```

throws the error.

---

## Fix 1 (Recommended)

Return only the array from the service.

```ts
// product.service.ts

import api from "@/src/libs/axios";
import { Product } from "../types/product";

export const getProducts = async (): Promise<Product[]> => {
  const response = await api.get("/products");

  return response.data.data; // ✅ only the array
};
```

Now your hook remains exactly the same:

```ts
useEffect(() => {
  getProducts().then(setProducts);
}, []);
```

and

```ts
products.map(...)
```

works correctly.

---

## Fix 2 (Alternative)

Keep the service unchanged:

```ts
return response.data;
```

Then update the hook:

```ts
useEffect(() => {
  getProducts().then((res) => {
    setProducts(res.data);
  });
}, []);
```

---

## Even better: Type your API response

Create a type like this:

```ts
export interface ApiResponse<T> {
  success: boolean;
  message: string;
  data: T;
}
```

Then your service becomes:

```ts
export const getProducts = async (): Promise<Product[]> => {
  const response = await api.get<ApiResponse<Product[]>>("/products");

  return response.data.data;
}
```

TypeScript will then prevent this kind of mistake because it knows the API returns an object with a `data` property rather than an array directly.

### One more thing

Your `Image` component is missing required dimensions unless you're using `fill`. It should be something like:

```tsx
<Image
  src={product.imageUrl}
  alt={product.name}
  width={400}
  height={300}
  className="h-56 w-full object-cover"
/>
```

or use `fill` inside a relatively positioned container.

The **`products.map is not a function`** error, however, is caused by returning `response.data` instead of `response.data.data` from your service.
